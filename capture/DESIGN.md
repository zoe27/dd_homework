# 钉钉客户端自动抓取方案设计

## 背景

钉钉 Stream SDK Bot 无法接收家校本专属群的消息，因此改为直接操控本机钉钉客户端，
通过截图 + OCR 的方式自动抓取作业内容。

---

## 模块结构

```
capture/
  find_window.py    找到钉钉主窗口，获取坐标和 window ID
  screenshot.py     按 window ID 截图；激活窗口
  ocr.py            easyocr 中文 OCR，返回带屏幕逻辑坐标的结果列表
  scraper.py        主流程（AI家校本入口版）
  scraper_scroll.py 旧版（群聊滚动扫描，已废弃，保留备用）
```

## 当前方案：AI家校本入口

### 流程概览

```
激活钉钉窗口，进入目标群聊
  ↓
步骤1：OCR 识别底部导航栏，找到"AI家校本"按钮并点击
  ↓
步骤2：等待面板渲染，点击"全部"tab（查看所有作业，而非只看未完成）
  ↓
步骤3：滚动扫描作业列表，收集所有卡片的唯一标识（日期+科目+发布人）
  第一次 OCR 识别当前可见卡片，记录每张卡片的 card_key
    ↓
  向下滚动面板，再次 OCR，追加新出现的 card_key
    ↓
  重复直到本轮识别到的所有 card_key 均已存在（无新卡片），或达到 MAX_HOMEWORK 上限
    ↓
  滚回列表顶部
  ↓
步骤4：按顺序循环处理步骤3收集到的每张卡片（最多 MAX_HOMEWORK 个）：
  OCR 当前列表，用 card_key 定位目标卡片：
    先匹配标题行（日期+科目），再验证其下方紧邻行的发布人是否与 card_key 一致
    两者均匹配 → 找到，取标题行坐标
    ↓
  若未找到（卡片不在当前可见区域）：
    滚回顶部 → 重新 OCR → 逐步向下滚动直到找到为止
    若仍找不到 → 跳过该卡片，记录警告
    ↓
  找到后检查卡片 y 坐标是否在安全区内（距面板顶部/底部各留 CARD_SAFE_MARGIN px）
    太靠近底部 → 小幅向下滚动，让卡片移到中间区域，重新 OCR 取新坐标
    太靠近顶部 → 小幅向上滚动，同上
    ↓
  点击卡片
    ↓
  等待 1.5s 渲染
    ↓
  截图右侧面板 → OCR 提取内容
    ↓
  检查是否有"∨"展开按钮 → 有则点击，等待 0.5s，再 OCR
    ↓
  在详情面板内向下滚动，每次滚动后对比截图像素差
  重复直到内容不再变化，或已达 MAX_DETAIL_SCROLLS 上限
    ↓
  合并所有 OCR 文字（按行去重）作为完整作业内容
    ↓
  点击返回按钮"<"（点击行左端 +10px，精准落在箭头上）
    ↓
  等待 0.8s，处理下一张卡片
  ↓
步骤5：关闭家校本面板
  优先：OCR 找到"×"按钮点击
  fallback：点击家校本面板左侧外部区域关闭（利用钉钉点击浮层外部关闭的交互）
  ↓
返回 RawMessage 列表 → card_parser → 生成PDF → 打印
```

## 坐标系说明

这是整个方案最关键的细节，涉及三层坐标转换：

```
截图（物理像素，Retina 2x = 逻辑尺寸 × 2）
  ↓ 裁剪右侧面板（物理像素偏移）
  ↓ 传入 easyocr，返回 bbox（物理像素）
  ↓ ÷ scale（2.0）→ 逻辑像素
  ↓ + panel_x_logical（面板裁剪偏移，逻辑）
  ↓ + window["x/y"]（窗口在屏幕上的位置，逻辑）
  = OcrResult.x/y（屏幕逻辑坐标）✓
  → 直接传给 CGEventCreateMouseEvent 点击 ✓
```

关键：`window["width/height"]` 是逻辑尺寸，截图是物理尺寸，`scale = 截图宽 / 窗口逻辑宽`。

## 各步骤技术细节

### 激活窗口
`NSWorkspace.sharedWorkspace().runningApplications()` 找到钉钉进程，
`app.activateWithOptions_(NSApplicationActivateIgnoringOtherApps)` 带到前台。
每次点击前都激活，确保事件发到钉钉。

### 截图
`Quartz.CGWindowListCreateImage` 按 window ID 截图，不要求窗口在最前面。
只截右侧面板区域（窗口宽度 37% 处往右），减少 OCR 处理量。

### OCR
`easyocr`，语言 `['ch_sim', 'en']`，置信度阈值 0.3。
首次加载模型约需几秒，之后复用单例。

### 点击
`CGEventCreateMouseEvent`，mouseDown + mouseUp 模拟单击。
坐标已经过 scale 转换，直接使用 OcrResult.center_x/y 或 x+offset。

### 步骤3：列表滚动扫描

卡片列表一屏只能显示 2-3 个，需要滚动才能获取全部。
同一科目可能有多个学生的作业（如两个孩子都有数学），因此不能只用科目名去重。

**卡片唯一标识（card_key）**：`日期 + 科目 + 发布人`，例如 `"4月14日数学_朱丹发布"`。
- 日期+科目：从标题行 OCR 识别（`RE_CARD_TITLE` 匹配）
- 发布人：标题下方的 `"XXX发布"` 行，OCR 识别后提取
- 若发布人识别失败，退而用 `日期 + 科目 + 摘要首行前10字` 作为 key

**扫描流程**：
- 扫描阶段收集 card_key 有序列表，不记坐标
- 每次滚动后 OCR，提取当前可见的所有 card_key：
  - card_key 必须完整（日期+科目+发布人 三者均识别到）才收录
  - 标题识别到但发布人缺失（卡片被截断只露出上半部分）→ 暂时忽略，下一轮滚动后该卡片会完整显示再收录
  - 已在列表中的 → 跳过（上下屏重叠的正常现象）
  - 不在列表中且完整的 → 追加
- 终止条件：本轮 OCR 识别到的所有完整 card_key 均已在列表中（无新卡片），或已达 MAX_HOMEWORK 上限
- 扫描完成后滚回顶部，再进入步骤4逐个处理

滚动方式：`CGEventCreateScrollWheelEvent`，在面板中心位置发送滚轮事件。

### 步骤4：循环定位与内容提取

**卡片定位逻辑**（解决坐标失效问题）：

步骤3收集的是卡片标题文字列表（有序），步骤4按顺序逐个处理。
每次处理第 i 张卡片时：
1. OCR 当前列表，查找目标标题文字
2. 找到后，检查卡片 y 坐标是否在面板安全区内（距顶部和底部各留 CARD_SAFE_MARGIN px）：
   - y 坐标太靠近底部边缘（卡片可能被截断）→ 小幅向下滚动，让卡片移到中间区域
   - y 坐标太靠近顶部边缘 → 小幅向上滚动
   - 在安全区内 → 直接点击
3. 找不到（卡片滚出可见区域）→ 滚回顶部，重新 OCR，再逐步向下滚动查找
4. 仍找不到 → 跳过，记录警告，继续下一张

这样无论列表当前滚动到哪个位置，都能可靠定位目标卡片。

详情页存在两种内容截断情况：

### 步骤4（续）：详情页完整内容提取
- OCR 识别到 `∨` 或 `展开` 关键词 → 点击
- 等待 0.5s 渲染 → 重新 OCR

**情况B：内容超出一屏（需要滚动）**
- 在详情面板区域内向下滚动
- 每次滚动后截图，与上一次截图对比底部区域像素差
- 终止条件：连续两次截图底部像素差低于阈值（内容不再变化），或已达 MAX_DETAIL_SCROLLS 上限
- 合并所有轮次的 OCR 文字，去除重复行后作为完整作业内容

注意：`去打印`、`反馈完成时长` 等按钮固定在底部，始终可见，不能作为终止判断依据。

过滤规则（skip_keywords）保持不变：
`去补交`、`有疑问`、`去打印`、`优秀作答`、`如何被选`、`排行榜`、`已完成`、
`预计需`、`已截止`、`昨日`、`首页`、`错题本`

### 返回按钮

匹配优先级（从高到低）：
1. 文字以 `<` 或 `〈` 开头，且包含 `的练习`（如"< 周沐曦的练习"）→ 最精准
2. 文字以 `<` 或 `〈` 开头（单独箭头行）
3. 文字包含 `的练习`（OCR 未能识别出 `<` 时的兜底）

找到后点击 `r.x + 10`（行左端），而非 `center_x`，确保落在箭头上而非文字中间。
若以上均未找到 → 点击面板左上角固定位置（`panel_x + 30, window_y + 30`）作为最终 fallback。

### 步骤5：关闭面板
优先 OCR 找 `×` 点击（同时兼容 OCR 误识别为 `X` 的情况）。
找不到时，点击家校本面板左侧外部区域：
- 点击坐标：`x = window_x + panel_x_logical / 2`，`y = window_y + window_height / 2`
- 利用钉钉"点击浮层外部自动关闭"的交互特性

## 参数配置

| 参数 | 值 | 说明 |
|---|---|---|
| CARD_OPEN_WAIT | 1.5s | 点击卡片后等待面板渲染 |
| EXPAND_WAIT | 0.5s | 点击"∨"展开后等待渲染 |
| SCROLL_WAIT | 0.8s | 每次滚动后等待渲染 |
| MAX_HOMEWORK | 6 | 最多提取作业数量 |
| MAX_DETAIL_SCROLLS | 5 | 详情页最多滚动次数（防止死循环） |
| CARD_SAFE_MARGIN | 60px | 点击前卡片距面板顶部/底部的最小安全距离（逻辑像素） |
| panel_x_ratio | 37% | 右侧面板起始位置（窗口宽度比例） |
| min_conf | 0.3 | OCR 最低置信度 |

## 与现有代码的衔接

`scraper.scrape()` 返回 `list[RawMessage]`，与 `card_parser.parse_messages()` 接口完全兼容。

```python
from capture.scraper import scrape
from parser.card_parser import parse_messages, sort_cards

messages = scrape()
cards = parse_messages(messages)
cards = sort_cards(cards)
# → 生成PDF → 打印，流程不变
```

## 风险与限制

- 需授权「屏幕录制」和「辅助功能」权限
- 钉钉需处于目标群聊界面，底部能看到"AI家校本"按钮
- 钉钉界面更新后坐标比例或文字可能变化，需重新调整
- 最小化状态下无法截图
- easyocr 首次加载模型需要几秒，后续复用
- 详情页滚动合并文字时，OCR 可能对同一行识别两次，需按行去重
